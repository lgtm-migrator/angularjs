Recent Searches API

Description

Search History tells you what terms users are searching for in applications. It helps the developers to maintain word cloud as well as it has the best support for data analytics. Word clouds have recently become a staple of data visualization. They are especially popular when analyzing text.

API Prerequisite

Using this API certainly doesn't require any JSON request payload. Everything the payload contains is optional. A summary of the payload values is described below;

  • size - Size parameter is passed to tell the wrapper, what size of data an user is expecting. If size is not added into the payload, query will retrieve 10 recent search records by default.
  • username - By default, elasticsearch Wrapper will give you result set of recent search history without any specification like which user made such searches. username parameter will help you get more refined search. When this parameters is passed to the elasticsearch, elasticsearch will serve you with the particular user dataset.
  • appname - A similar approach like username, appname will get the application specific search made by an user if username key is passed, else a complete dataset of recent searches made by authorized users onto the application.

Request Sample

NOTE: Recent Search History is disabled by default on all the applications except GlobalLessonLearnt. It can be enabled/disabled on application level. Application owner must ask developers to make it active on their application configuration.

Response Sample

What we preserve in Search History?

The following information is captured from user session and kept into the elasticsearch index to maintain search history data.

  • username - used to track the authenticated user to the application.
  • appname - the application name user has accessed.
  • key- the search term user has requested.
  • timecreated- the timestamp at which user initiate the search.
  • devicetype - and the device type user was using to access the application.
